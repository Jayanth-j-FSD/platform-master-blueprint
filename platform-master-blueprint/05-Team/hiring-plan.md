# TEAM ROLES & HIRING PLAN
This document defines the complete team structure, roles, responsibilities, and hiring phases for building a world-scale identity + social + marketplace + collaboration + payments platform.

The goal:  
Hire lean early → build the core → expand with demand → scale with traction.

---

# 1. PHASE OVERVIEW
Your hiring must happen in **4 phases**:

### **Phase 0 — Founder-Only (0 → Prototype)**
People involved:
- Founders only (3–5 max)
- Maybe 1 contractor for UI or backend support

Target timeline: 1–3 months  
Goal: Build the **Identity Layer**, internal admin tools, and platform skeleton.

### **Phase 1 — MVP Team (4 → 12 people)**
Focused on:
- Identity Layer
- Social base layer
- Marketplace scaffolding
- Payments integration (basic)
- QA + security baseline

### **Phase 2 — Growth & Network Effects Team (12 → 30 people)**
New teams:
- Growth
- Data
- Trust & Safety
- Support
- Product expansion teams

### **Phase 3 — Scale Team (30 → 100+)**
Add:
- Advanced AI/ML teams
- Enterprise team
- Multi-country operations
- Finance, compliance, infra scale

# 3. PHASE-WISE HIRING PLAN

---

# PHASE 0 — FOUNDER-ONLY TEAM (0–3 MONTHS)

### **Required Core Founders**
1. **CTO / Lead Architect**
   - Owns entire architecture (identity + payments + marketplace)
2. **Chief Product Engineer**
   - Builds frontend + API integration
   - Defines UX + flows
3. **Backend Lead (Founder or early employee)**
   - Owns NestJS, microservices, DB schemas, integrations
4. **Tech Generalist / Multirole Engineer**
   - Acts as FE/BE/mobile as needed

### Optional (Contract)
- UI/UX designer (1–2 months)
- DevOps consultant (once for infrastructure setup)

**Total headcount: 3–5**

---

# PHASE 1 — MVP TEAM (MONTH 3–9)

### ENGINEERING (MUST-HAVE)
1. **Backend Engineer — Identity**
   - Auth, verification, trust graph
2. **Backend Engineer — Marketplace/Social**
   - Listings, feed, search
3. **Frontend Engineer**
   - React + mobile integration
4. **Mobile Engineer (React Native)**
   - Lightweight, MVP-focused
5. **DevOps / SRE Engineer**
   - Kubernetes, infra automation
6. **Security Engineer**
   - Threat model + identity hardening

### PRODUCT
1. **Product Designer (UX/UI)**

### QA
1. **QA Engineer** (automation preferred)

**Total headcount: 8–12**

---

# PHASE 2 — GROWTH TEAM (MONTH 9–18)

### ENGINEERING
- **3 more Backend Engineers** (marketplace, social graph, payments)
- **2 Frontend Engineers** (scaling UI, dashboards)
- **1 Mobile Engineer**
- **1 DevOps** (2 total)
- **1 SDET** (test automation expert)

### DATA & AI
- **Data Engineer**
- **ML Engineer** (recommendations, trust scoring)
- **Data Analyst**

### TRUST & SAFETY
- **Trust & Safety Engineer**
- **Fraud Analyst**
- **Compliance Officer**

### PRODUCT & GROWTH
- **Product Manager**
- **Growth Lead**
- **Content strategist**

**Total headcount: ~20–30**

---

# PHASE 3 — SCALE TEAM (18–36 MONTHS)

At 1M → 10M users, you need:

### ENGINEERING
- **Team leads for each layer**  
  - Identity  
  - Social  
  - Marketplace  
  - Payments  
  - Collaboration  
- **Senior Platform Engineer**
- **Site Reliability Engineers**
- **Database Performance Engineer**
- **Full AI Division (3–6 people)**

### BUSINESS
- **Finance Team**
- **Legal Team**
- **Country Expansion Managers**

### TRUST & SAFETY
- **Moderation team**
- **Fraud prevention operators**
- **24/7 support leads**

### SUPPORT
- **Customer success team**
- **Technical support team**

**Total headcount: 50–120+**

# 4. ROLE DEFINITIONS (FULL RESPONSIBILITIES)

---

# ENGINEERING ROLES

## **1. Backend Engineer — Identity**
Responsibilities:
- Build identity, auth, session, and verification systems
- Manage trust graph & scoring pipeline
- Secure user PII handling
- Build high-availability APIs
- Lead zero-trust access model enforcement

## **2. Backend Engineer — Social**
- Build posts, feed, messaging
- Search + content ranking
- Graph-based social features

## **3. Backend Engineer — Marketplace**
- Order system, listings, reviews
- Search, pricing, fraud checks
- Escrow integration with payments team

## **4. Backend Engineer — Payments**
- Wallet, ledger, transactions
- Settlement, fees, reconciliation
- Fraud detection and AML rules

## **5. Frontend Engineer**
- React web app
- Component system design
- Identity, social, and marketplace interfaces

## **6. Mobile Engineer**
- React Native app
- Offline caching, real-time sync
- Push notifications

## **7. DevOps Engineer**
- Kubernetes, CI/CD
- Monitoring & alerting
- Auto-scaling & cost optimization

## **8. Security Engineer**
- Threat modeling
- Penetration tests
- Abuse detection
- Secure code reviews

---

# PRODUCT ROLES

## **9. Product Manager**
- Defines what gets built
- Works with engineering to plan roadmap
- Owns onboarding flows and retention metrics

## **10. Product Designer (UX/UI)**
- Full design system
- User flows for identity → social → marketplace → payments
- App layout, screens, branding alignment

---

# DATA ROLES

## **11. Data Engineer**
- Build pipelines, ETL, data lake
- Set up metrics and dashboards

## **12. Machine Learning Engineer**
- Trust scoring ML
- Recommendations
- Fraud and anomaly detection

## **13. Data Analyst**
- Insights, funnels, dashboards
- Cohort analysis and growth metrics

---

# TRUST & SAFETY ROLES

## **14. Fraud Analyst**
- Detect bad actors
- Monitor suspicious patterns
- Work with payments team

## **15. Trust & Safety Engineer**
- Policy enforcement automation
- Moderation tools
- User flagging systems

## **16. Compliance Officer**
- KYC/AML regulations
- Data privacy audits
- Country-specific compliance

---

# GROWTH & BUSINESS ROLES

## **17. Growth Lead**
- Run acquisition experiments
- Build referral loops
- Track LTV, CAC, retention metrics

## **18. Partnerships Manager**
- External integrations
- API partnerships
- Payment partner relations

## **19. Business Operations**
- Internal process building
- Cross-functional workflows
- Performance tracking

---

# SUPPORT ROLES

## **20. Customer Support**
- Handle user issues, disputes
- Maintain helpdesk

## **21. Technical Support**
- Debug issues
- Assist engineers
- Manage admin tools

# 5. HIRING PRIORITY ORDER (FROM MOST CRITICAL)

### Absolute must-hire first:
1. Backend Engineer — Identity  
2. Backend Engineer — Social/Marketplace  
3. Frontend Engineer  
4. DevOps  
5. Security Engineer  
6. Product Designer  

### Don't waste money early on:
- Marketing agencies  
- Sales team  
- HR  
- Finance  
- Content teams  
- Moderators (do automation first)

# 6. SALARY & EQUITY GUIDELINES (OPTIONAL)
(Adapt for country)

### Early Stage (0 → 12 people)
- Lower salaries  
- Higher equity  
- Target: 0.25%–1% for early engineers  
- Founders: locked via vesting

### Growth Stage (12 → 30)
- Balanced salary  
- Lower equity  
- Recruit senior specialists

### Scale Stage (30+)
- Market salary  
- Minimal equity  
- Team leads & managers
